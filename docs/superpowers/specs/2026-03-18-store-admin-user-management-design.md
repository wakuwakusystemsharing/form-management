# 設計書: 店舗管理者ユーザー管理機能拡張

**日付:** 2026-03-18
**対象ブランチ:** staging
**スコープ:** StoreAdminManager の機能拡張

---

## 概要

店舗管理者管理画面に以下3つの機能を追加する。

1. **ログインURLコピー** — 店舗管理者ログインページのURLをワンクリックでコピー
2. **ユーザー編集モーダル** — メールアドレス・パスワードの変更
3. **PATCH APIエンドポイント** — ユーザー情報更新のバックエンド

---

## 1. ログインURLコピー機能

### UI
- `StoreAdminManager` カードヘッダーの「ユーザー追加」ボタン横に「ログインURLをコピー」ボタン（`Copy` アイコン）を追加
- クリック時: クリップボードにURLをコピー → トースト通知「ログインURLをコピーしました」

### URLフォーマット
```
{baseUrl}/store/{storeId}
```
- `getBaseUrl()` 関数（`src/lib/env.ts`）を使用して環境ごとのベースURLを取得

---

## 2. ユーザー編集モーダル

### テーブル変更
- 操作列（現在: 削除ボタンのみ）に「編集」ボタン（`Pencil` アイコン）を追加
- 「編集」クリックで対象ユーザー情報をプリフィルした編集モーダルを表示

### モーダル内容
| フィールド | 説明 |
|-----------|------|
| メールアドレス | 現在値をプリフィル。変更可能 |
| 新しいパスワード | 空白の場合は変更しない |

- バリデーション: パスワードは6文字以上（入力した場合のみ）、メールは `type="email"` で形式チェック
- 保存中は保存ボタンを disabled にし「保存中...」と表示（`isSaving` フラグで管理）
- 保存成功時: テーブルのメールアドレス表示を更新 + トースト通知
- ステータス列・禁止機能は追加しない（削除→再作成の運用）

## 3. PATCH APIエンドポイント

### エンドポイント
`PATCH /api/stores/[storeId]/admins/[userId]`

### リクエストボディ
```json
{
  "email": "new@example.com",
  "password": "newpassword123"
}
```
両フィールドとも省略可。少なくとも1つが必要。

### 認証・認可フロー
1. local環境: `shouldSkipAuth()` が true のためスキップ
2. staging/production:
   - `auth-helper.ts` の `getCurrentUser()` でリクエストからユーザーを取得
   - 未認証の場合: 401 を返す
   - 認証済みユーザーが `isServiceAdmin()` でサービス管理者か確認
   - サービス管理者でない場合: 403 を返す

### 処理フロー
1. `email` と `password` が両方省略された場合: 400 を返す
2. `email` が指定された場合: Supabase `auth.admin.updateUserById()` でメール更新
   - メール重複時: Supabase エラーをキャッチし 409 で「このメールアドレスは既に使用されています」を返す
3. `password` が指定された場合: 同APIでパスワード更新（6文字未満は400で返す）
4. local環境: `data/store_admins_{storeId}.json` の `user_id` 一致レコードの `email` を更新
   - パスワード変更はスキップ
   - 対象レコードが見つからない場合: 404 を返す

### レスポンス
```json
// 成功
{ "success": true, "email": "new@example.com" }

// エラー
{ "error": "日本語のエラーメッセージ" }
```
