# 公開HP・プライバシーポリシー・利用規約 設計書

**日付**: 2026-03-19
**目的**: Google OAuth審査に必要な公開ページ（ホームページ、プライバシーポリシー、利用規約）の追加

## 背景

Google OAuthの本番公開（restricted scope verification）には以下が必要:
- アプリケーションのホームページ
- 一般公開のプライバシーポリシーへのリンク
- 一般公開の利用規約へのリンク

現在これらのページは存在しないため、Next.jsアプリ内に静的ページとして追加する。

## サービス情報

- **サービス名**: NAS (Need Appointment System)
- **運営者情報**: 仮値（プレースホルダー）で実装。本番公開前に実値に差し替え

## ページ構成

### ルーティング

| パス | ページ | 認証 |
|------|--------|------|
| `/home` | ホームページ | 不要 |
| `/privacy` | プライバシーポリシー | 不要 |
| `/terms` | 利用規約 | 不要 |

- 現在の `/` → `/admin` リダイレクトは維持
- `(public)` ルートグループで共通レイアウトを適用
- 現在の middleware.ts は `/admin` と店舗管理パスのみ保護しているため、これらのパスは変更なしで公開される
- Google OAuth コンソールには `https://domain.com/home` をホームページURLとして登録

### 共通レイアウト `(public)/layout.tsx`

- **ヘッダー**: サービス名「NAS」+ ナビリンク（ホーム、プライバシーポリシー、利用規約）
- **フッター**: 運営者情報、各ページリンク、コピーライト `© 2026 [運営者名]`
- Tailwind + shadcn/ui で既存デザインシステムを活用
- モバイルレスポンシブ対応

### ホームページ `/home`

**構成:**
1. **ヒーローセクション**: サービス名 + キャッチコピー「LINE連携の予約フォーム管理システム」+ 簡潔な説明2〜3行
2. **機能紹介セクション**: shadcn/ui Card × 4枚
   - 予約フォーム作成・管理
   - LINE LIFF連携
   - Google Calendar連携
   - 顧客管理（CRM）
3. **フッター**: 共通レイアウトのフッター

**デザイン**: 最小限の装飾。既存テーマカラー（CSS変数）使用。

### プライバシーポリシー `/privacy`

個人情報保護法（2022年改正）準拠。以下の項目を含む:

1. **個人情報取扱事業者の名称・連絡先** — `[運営者名]` `[所在地]` `[メールアドレス]` （プレースホルダー）
2. **取得する個人情報の種類と取得方法**:
   - 氏名、電話番号、メールアドレス — 予約フォーム送信時に直接取得
   - LINE ユーザーID — LINE LIFF SDK経由で自動取得
   - 予約情報（メニュー、日時） — 予約フォーム送信時に直接取得
   - Google Calendarデータ — OAuth同意に基づきGoogle Calendar API経由で取得
3. **利用目的**:
   - 予約管理・サービス提供
   - ユーザーへの連絡
   - サービス改善・分析
4. **第三者提供**:
   - 原則行わない
   - 法令に基づく場合を除く
   - Google Calendar連携はユーザー同意のもと
5. **業務委託・外国にある第三者への提供**:
   - Supabase, Inc.（米国 — データベースホスティング）
   - Vercel, Inc.（米国 — Webホスティング）
   - LINEヤフー株式会社（日本 — メッセージング）
   - 各委託先は適切なデータ保護措置を講じている旨を記載
   - 米国はAPPI上の「個人の権利利益を保護する上で我が国と同等の水準にある国」に該当しないため、その旨を開示
6. **保有個人データの開示・訂正・追加・削除・利用停止・消去請求**: ユーザーの5つの権利と請求方法（メールで受付）
7. **個人データの保存期間**: アカウント削除後1年間保持、その後削除。Google Calendarデータは連携解除時に即時削除
8. **安全管理措置**:
   - 組織的措置: 個人情報管理責任者の設置、アクセス権限管理
   - 技術的措置: SSL/TLS暗号化、Row Level Security、アクセス制御
   - 物理的措置: クラウドサービス（Supabase/Vercel）のデータセンターセキュリティに依拠
9. **Cookie・アクセスログ**: Vercel Speed Insights使用の旨
10. **Google APIサービスの利用**:
    - Google Calendar APIで取得する情報の範囲（カレンダーイベントの読み書き）
    - 利用目的（予約のカレンダー同期のみ）
    - Google API Services User Data Policy への準拠
    - **Limited Use ディスクロージャー**: Google APIから受信した情報の使用および他のアプリへの転送は、制限付き使用の要件（Limited Use requirements）を含むGoogle APIサービスのユーザーデータに関するポリシーに準拠する旨を明記
    - Google ユーザーデータを広告目的に使用しない旨
    - サービス提供に必要な場合を除き、Google ユーザーデータを第三者に転送しない旨
11. **改定**: 重要な変更時はサービス上で通知
12. **制定日**: 2026年3月19日

### 利用規約 `/terms`

日本法準拠。以下の項目を含む:

1. **総則**: NASの定義、規約の適用範囲
2. **利用登録**: 店舗管理者の登録条件
3. **アカウント管理**: ID・パスワードの管理責任
4. **禁止事項**: 不正利用、法令違反、サーバー負荷、逆コンパイル等
5. **サービスの変更・停止・終了**: 事前通知と免責
6. **知的財産権**: サービスの著作権は運営者に帰属
7. **免責事項**: 損害賠償の制限、予約トラブルへの非関与
8. **個人情報の取扱い**: プライバシーポリシーへの参照リンク
9. **準拠法・管轄裁判所**: 日本法準拠、東京地方裁判所（仮）
10. **制定日**: 2026年3月19日

## 技術仕様

- **フレームワーク**: Next.js App Router
- **スタイリング**: Tailwind CSS v4 + shadcn/ui (Card コンポーネント)
- **レンダリング**: 静的ページ（SSG可能）
- **認証**: 不要（middleware.ts で公開パスとして設定）
- **SEO**: 各ページに適切な `<title>` と `<meta description>` を設定
- **言語**: `(public)/layout.tsx` で `lang="ja"` を設定（ルートlayoutは `lang="en"` のため上書き）
